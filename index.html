JQ::Lite は純Perlで動く軽量JSONクエリ＆編集ツール。
制約の多い企業環境でも、jq と同等の体験をそのまま実現します。

JSON Playground
$ perl -MJQ::Lite -E 'say jq(q|.users[]| => { users => [ { name => "Alice" }, { name => "Bob" } ] })'
{"name":"Alice"}
{"name":"Bob"}

$ jq-lite users.json '.stats | {active, lastSync}'
{
  "active": true,
  "lastSync": "2025-03-12T09:44:00"
}

jq の書き心地のまま、制限付きサーバに導入可能。日常的な運用保守にフィットするツールです。

Why JQ::Lite?

制約のある現場でこそ真価を発揮

レガシーサーバ、閉域網、厳格な社内ルール。JQ::Lite は、そんな「入れ替えできない」現場のために設計されています。

🧬

純Perlワンファイル構成

CPAN モジュールの追加も、ビルドも不要。環境にそのままドロップインできます。

⚡

軽量＆高速

ミニマルな実装で、制約された CPU / メモリ環境でも快適に動作します。

🛡️

依存ゼロで安全

コンパイル不要・追加バイナリ不要。セキュリティポリシーが厳しい現場でも安心です。

🧰

CLI / モジュール両対応

既存の Perl スクリプトやシェルスクリプトに自然に組み込める柔軟性。

🔍

jq互換の直感操作

シンタックスは jq を踏襲。学習コストゼロで導入できます。

📦

テキスト処理にも最適

ログ整形、APIレスポンス解析、CI/CD レポート作成など幅広いワークフローで活躍します。

導入から活用までの流れ

JQ::Lite は環境構築に時間をかけたくない運用チームの味方です。たった3ステップですぐに JSON 処理を自動化できます。

  • 1

    ファイルを設置

    CPAN から cpanm で入れるか、単一ファイルを環境に配置するだけ。

  • 2

    フィルタを書く

    jq と同じ書き方で JSON を抽出・変換。既存のナレッジをそのまま活かせます。

  • 3

    スクリプトへ組み込む

    Perl モジュールとして require しても、CLI としてパイプに流し込んでも OK。

Config Snapshot

# JSON内の特定ユーザを抽出
$ jq-lite users.json '.users[] | select(.role == "admin") | {name, last_login}'

# Perl スクリプトから利用
use JQ::Lite qw/jq/;
my $json = decode_json($raw);
my $admins = jq('.users[] | select(.role == "admin")', $json);

ツールの導入障壁を極限まで下げたことで、セキュアな閉域網でも運用フローがシンプルになります。

現場の運用に溶け込むユースケース

痒いところに手が届く JSON 処理で、日々の保守作業をスマートに。

閉鎖ネットワークのシステム運用

外部バイナリを導入できない環境でも JSON 設定ファイルの整形・検証を自動化。

  • ・CI サーバなしでも差分を検証
  • ・バックアップ前に JSON の整合性をチェック

レガシー移行プロジェクト

既存 Perl スクリプトを活かしながら段階的に JSON ベースの運用へ移行。

  • ・API 応答を整形して旧システムへ受け渡し
  • ・ログを JSON へ変換して可視化基盤へ送信

CI/CD レポートの集約

ビルドログやテスト結果の JSON を一括処理して、Slack やメールに要約を投稿。

  • ・バッチ処理に jq-lite を組み込み
  • ・GitHub Actions から直接 Perl スクリプトを実行

監視・アラートの自動化

監視ツールが出す JSON を整形して、閾値超過時のみ通知するなど柔軟なワークフローを構築。

  • ・サーバリソースの JSON レポートを要約
  • ・既存の shell スクリプトと相互運用
Install

最小手順で導入

環境制限の厳しいオンプレミスサーバでも、3 分で導入完了。CI/CD や cron バッチにもそのまま活用できます。

✅ Perl 5.10+ ✅ 外部依存ゼロ ✅ 単一バイナリ 
# CPAN からインストール
cpanm JQ::Lite

# 単一ファイルで運用
curl -O https://raw.githubusercontent.com/kawamurashingo/JQ-Lite/main/bin/jq-lite
chmod +x jq-lite

# JSONの加工例
jq-lite users.json '.users | map(select(.active)) | length'
